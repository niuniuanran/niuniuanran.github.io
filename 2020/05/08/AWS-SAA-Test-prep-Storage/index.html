S3 Security

Access Control

All new buckets are PRIVATE when created by default.
Logging can be turned on a bucket to log all read/write histories. Logging will be saved in a different bucket (can even belong to another account.)

Bucket Policies

Simple way of granting access

Access Control Lists (ACL)

JSON object, more refined, complex rule access.

Encryption

Encryption in Transit

Local <-> S3 is encrypted via SSL/TLS

Encryption At Rest: Server Side Encryption

Amazon can help you encrypt the object data: S3 managed keys (amazon manages all keys)

  • SSE-AES: S3 handles the key, uses the AES-256 alg.
  • SSE-KMS: Envelope encryption, AWS KMS and you manage the keys together
  • SSE-C: Customer provided key (you manage the keys yourself)

    Client-Side Encryption

    You encrypt your own files before uploading to S3.

S3 Data Consistency

New Objects (PUTS)

Read after Write Consistency: when you upload a new S3 object you are able to read immediately after writing.

Overwrite(PUTS) or Delete Objects (DELETES)

Eventual consistency
When you overwrite or delete an object, it takes time for S3 to replicate versions to AZs
If you were to read immediately, S3 may return you an old copy.

S3 Cross-Region Replication (CRR)

Provides higher durability and potential disaster recovery for objects.

You must have versioning turned on both the source and destination buckets.

S3 Versioning

Versioning allows you to help prevent data loss and keep track of history versions.
New versions of file default to be private, even if its older version is made public.

Fully integrates with S3 Lifecycle rules
Store all versions of an object in S3.
MFA delete feature provides extra protection against deletion of data.

Once you turn on versioning, you cannot remove it. You can suspend versioning

Lifecycle Management

Automate the process of moving objects to different storage classes or delete objects all together.

Can be used together with versioning.
Can be applied to both current and previous versions.

S3 Transfer Acceleration

Keywords:

  • CloudFront
  • Edge location
  • direct URL
  • AWS Backbone Network connecting Edge location <-> bucket

Presigned URLs

Presigned URLs are commonly used to provide access to private objects
Use:

  • AWS CLI
  • AWS SDK
    to generate presigned URLs

Example usage:
You have a web-application that needs to allow user to download a file from a password-protected part of your web-app. Your web-app generates the Presigned URL which expires in 5 seconds to let the user download the file.

MFA Delete

In order to enable MFA:

  • AWS CLI must be used
  • versioning must be enabled.
  • Only bucket owner logged in as Root User can DELETE objects from bucket.

Region

The S3 interface is global.
The bucket still belongs to a region.

S3 objects are automatically duplicated at 3 AZs, except S3 One Zone-IA.

EBS is automatically duplicated within the one AZ.
You can enable Multi-AZ for RDS, and Global Table for multiple regions.
Multi-AZ deployments for MariaDB, MySQL, Oracle, and PostgreSQL DB instances use Amazon’s failover technology. Microsoft SQL Server DB instances use SQL Server Database Mirroring (DBM) or Always On Availability Groups (AGs).
EC2 instance/ subnet belongs to one AZ, one region. VPC belong to one region.
IAM is global.

SSA S3 Summary

  • Simple Storage Service(S3) is an object-based storage.

  • Store unlimited amount of data without worry of storage infrastructure: fully managed

  • S3 (except S3 One zone- Infrequent Access) replicates data across 3 AZs to ensure 99.99% availability and 11’ 9s durability.

  • Objects contain your data (like files), size can be from 0 byte to 5 TB.

  • Buckets contain objects. Buckets can also contain folders that contain objects

  • Bucket names are unique across all AWS accounts, like a domain name.

  • When you upload a file to S3 successfully, receive a HTTP 200 response code.

  • Lifecycle Management: Objects can be moved between storage classes/ deleted automatically based on a schedule.

  • Versioning: Objects are given a version ID. When new objects are uploaded the old versions and kept and you can access any object version. Once turned on versioning can only be suspended. When you delete an object the previous object is restored.

  • MFA delete: enforce DELETE operations to require MFA token in order to delete an object. Must have versioning turned on. You can only turn on MFA from AWS CLI. If a bucket’s versioning configuration is MFA Delete–enabled, the bucket owner must include the x-amz-mfa request header in requests to permanently delete an object version or change the versioning state of the bucket. Requests that include x-amz-mfa must use HTTPS.

  • All new buckets are private by default.

  • Logging can be turned on for a bucket to track operations performed on objects, logging saved in another S3 bucket (even of a different account)

  • Access Control is configured using bucket policies and access control lists. Amazon S3 access control lists (ACLs) enable you to manage access to buckets and objects. Each bucket and object has an ACL attached to it as a subresource.

  • Security in transit: SSL

  • SSE(server side encryption): SSE-KMS(envelope encryption), SSE-AES(S3 manages the key), SSE-C(customer key)

  • CCR(cross region replication) allows you to replicate files across regions. You must have versioning turned on in the source and destination bucket. The two buckets can belong to different AWS account.

  • Transfer Acceleration: data is uploaded via distinct url to an edge location, and transported to S3 bucket via AWS backbone network.

  • Presigned URLs: generated via AWS CLI or SDK. commonly used to grant access to private objects.

Snowball, Snowball Edge, Snowmobile

Export/import data
into S3/Glacier

  • Snowball and Snowball Edge is a rugged container which contains a storage device. For peta-scale (fifth power of 1000) migration.
    • Snowball comes in two sizes: 50TB, 80TB
    • Snowball Edge comes in two sizes: 100 TB, 100TB Clustered
  • Snowmobile is a 45-foot long shipping container, pulled by a semi-trailer truck. For exabyte-scale (sixth power of 1000) migration. Comes in one size: 100PB.

Snowball Edge

  • Can undertake local processing and edge-computing workloads
  • Can use in a cluster in groups of 5 to 10 devices
  • Provides three options for device configurations:
    • Storage optimized (24 vCPUs)
    • Compute optimized (54 vCPUs)
    • GPU optimized (54 vCPUs)
