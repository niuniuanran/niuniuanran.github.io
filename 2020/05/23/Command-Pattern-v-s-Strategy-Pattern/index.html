Command Pattern: “Encapsulating Invocation”

Problem

  • 在某些场合，比如要对行为进行”记录、撤销/重做、事务”等处理，这种无法抵御变化的紧耦合是不合适的。
  • 在这种情况下，如何将”行为请求者“与”行为实现者“解耦？
  • 将一组行为抽象为对象，可以实现二者之间的loose coupling。

Roles

Three key roles:

  1. Command
  2. Receiver: who the command works ON
  3. Invoker: the entry point of using the command.

Benefits

  1. Loose Coupling between invoker and receiver
  2. Easier to implement and add new commands
