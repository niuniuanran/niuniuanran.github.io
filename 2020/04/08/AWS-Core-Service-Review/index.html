VPC

  • Description: Create a virtual network
  • What sort of service does it provide: Networking
  • Non-cloud equivalent: local network
  • IaaS/PaaS/SaaS/ does not apply: IaaS/Does not apply
  • AWS Managed? Yes. You have to configure some parts, but AWS manages the virtual border of your network.
  • Related Services NAT Gateway, Internet Gateway, EC2, EBS, etc.

NAT Instance and NAT Gateway

AWS’s comparison of NAT instance vs NAT Gateway

NAT Instance

  • Description: An EC2 instance that does network address translation. It lies in a public subnet in your VPC, to enable instances in the private subnet to initiate outbound IPv4 traffic to the Internet or other AWS services

    Might not always be available because of your instance shutdown, but NAT Gateway is always available; The bandwidth of NAT gateway depends on your instance bandwidth, and NAT Gateway can scale up to 45 Gbps
    Could be cheaper than NAT gateway

  • What sort of service does it provide: Networking, address translation
  • Non-cloud equivalent: Physical NAT service
  • IaaS/PaaS/SaaS/ does not apply IaaS
  • AWS Managed? No, user managed. You are using an EC2 and configuring your service
  • Related Services EC2, RDS, DynamoDB

NAT Gateway

  • Description: Functions similarly as NAT instance. Network address translation (NAT) gateway to enable instances in a private subnet to connect to the internet or other AWS services, but prevent the internet from initiating a connection with those instances. Provides more availability and bandwidth
  • What sort of service does it provide: Networking, address translation
  • Non-cloud equivalent: Physical NAT service
  • IaaS/PaaS/SaaS/ does not apply SaaS. You are consuming it like a software
  • AWS Managed? Yes
  • Related Services EC2, RDS, DynamoDB

Internet Gateway

  • Description: An internet gateway is a horizontally scaled, redundant, and highly available VPC component that allows communication between instances in your VPC and the internet.
    It therefore imposes no availability risks or bandwidth constraints on your network traffic.
    • Enabling Internet Access to/from instances in a VPC subnet:
      • Attach an Internet Gateway,
      • Add a route to direct Internet-bound traffic to the Internet Gateway;
      • Ensure instances have a unique public IP (public IPv4, Elastic IP, IPv6 address)
      • Ensure the network access control lists (NACL) and security group rules allow the relevant
  • What sort of service does it provide: Networking
  • Non-cloud equivalent: Router
  • IaaS/PaaS/SaaS/ does not apply IaaS
  • AWS Managed? Yes
  • Related Services VPC, EC2

CloudWatch

  • Description: CloudWatch collects monitoring and operational data in the form of logs, metrics, and events

    Overall system performance measures: number of requests, traffic load, I/O stuff. Like from outside looking in: how does the system behave, CPU usage, etc.

  • What sort of service does it provide: Security
  • Non-cloud equivalent: log
  • IaaS/PaaS/SaaS/ does not apply SaaS
  • AWS Managed? Yes
  • Related Services Lambda, EC2, EBS, S3, EFS

CloudTrail

  • Description: With CloudTrail, you can log, continuously monitor, and retain account activity related to actions across your AWS infrastructure.

    Records accounts’ behaviours, access to resources. Allows you to go into more details, like who made what API calls and did what. Records the user that makes API calls.

  • What sort of service does it provide: Security
  • Non-cloud equivalent: log
  • IaaS/PaaS/SaaS/ does not apply SaaS
  • AWS Managed? Yes
  • Related Services IAM, AWS Organizations

EC2

  • Description: Virtual machine for multiple purposes of computing service
  • What sort of service does it provide: Computing
  • Non-cloud equivalent: Computer
  • IaaS/PaaS/SaaS/ does not apply IaaS
  • AWS Managed? No
  • Related Services EBS, VPC, S3, RDS, CloudWatch

Lambda

  • Description: It lets you run code as driven by pre-defined events, like a function without the need to configure a server
  • What sort of service does it provide: Computing
  • Non-cloud equivalent: Computer
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes
  • Related Services EC2, CloudWatch, IAM

Elastic Beanstalk

  • Description: AWS Elastic Beanstalk is an easy-to-use service for deploying and scaling web applications and services
    You can simply upload your code, and AWS Elastic Beanstalk automatically handles the deployment, from capacity provisioning, load balancing, and auto scaling to application health monitoring. At the same time, you retain full control over the AWS resources powering your application and can access the underlying resources at any time.
  • What sort of service does it provide: Compute
  • Non-cloud equivalent: Web server hosts like Xampp , Lampp
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Partially
  • Related Services EC2, VPC

Fargate

  • Description: AWS Fargate is a compute engine for Amazon ECS that allows you to run containers without having to manage servers or clusters.

    Amazon ECS has two modes: Fargate launch type and EC2 launch type. With Fargate launch type, all you have to do is package your application in containers, specify the CPU and memory requirements, define networking and IAM policies, and launch the application. EC2 launch type allows you to have server-level, more granular control over the infrastructure that runs your container applications.

Kubernetes: Manage a cluster of Linux containers as a single system to accelerate Dev and simplify Ops

  • What sort of service does it provide: Compute, container service launching
  • Non-cloud equivalent: Your own container orchestration software
  • IaaS/PaaS/SaaS/ does not apply Containers as a Service
  • AWS Managed? Yes
  • Related Services ECS, EC2

ECS

  • Description: Amazon Elastic Container Service (Amazon ECS) is a highly scalable, high-performance container orchestration service that supports Docker containers and allows you to easily run and scale containerized applications on AWS.
  • What sort of service does it provide: Computing
  • Non-cloud equivalent: Your own container orchestration software
  • IaaS/PaaS/SaaS/ does not apply Container as a Service
  • AWS Managed? Yes
  • Related Services Fargate, EC2

EKS

  • Description: Amazon Elastic Container Service for Kubernetes (Amazon EKS) makes it easy to deploy, manage, and scale containerized applications using Kubernetes on AWS.

    Amazon EKS runs the Kubernetes management infrastructure for you across multiple AWS availability zones to eliminate a single point of failure. Amazon EKS is certified Kubernetes conformant so you can use existing tooling and plugins from partners and the Kubernetes community. Applications running on any standard Kubernetes environment are fully compatible and can be easily migrated to Amazon EKS.

  • What sort of service does it provide: Compute
  • Non-cloud equivalent: ****
  • IaaS/PaaS/SaaS/ does not apply Container as a service
  • AWS Managed? Yes
  • Related Services ****

ECR

  • Description: Amazon Elastic Container Registry (Amazon ECR) is a fully-managed Docker container registry that makes it easy for developers to store, manage, and deploy Docker container images.

    Amazon ECR is integrated with Amazon Elastic Container Service (Amazon ECS), simplifying your development to production workflow. Amazon ECR eliminates the need to operate your own container repositories or worry about scaling the underlying infrastructure.

  • What sort of service does it provide: Computing
  • Non-cloud equivalent: ****
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes
  • Related Services ECS

S3

  • Description: Amazon Simple Storage Service (Amazon S3) is an object storage service that offers industry-leading scalability, data availability, security, and performance.
  • What sort of service does it provide: Storage
  • Non-cloud equivalent: Hard drive
  • IaaS/PaaS/SaaS/ does not apply PaaS
    • It’s not IaaS it’s a ready to go service, you can run it as is without the need of “configuring/building/installing” anything.
    • its more of a PaaS than SAAS because you can build so many things on top of S3 using APIs, for example the integration of cloudfront, encryption, static websites, lifecycle management etc.
  • AWS Managed? No
  • Related Services EC2

EBS

  • Description: Amazon Elastic Block Store (Amazon EBS) provides persistent block storage volumes for use with Amazon EC2 instances in the AWS Cloud.

    Amazon EBS volumes offer the consistent and low-latency performance needed to run your workloads. With Amazon EBS, you can scale your usage up or down within minutes—all while paying a low price for only what you provision.

  • What sort of service does it provide: Storage
  • Non-cloud equivalent: Hard drive
  • IaaS/PaaS/SaaS/ does not apply IaaS
  • AWS Managed? No
  • Related Services EC2

    EBS is a virtual hard disk in the cloud and serves as block storage

EFS

  • Description: Amazon Elastic File System (Amazon EFS) provides a simple, scalable, elastic file system for Linux-based workloads for use with AWS Cloud services and onpremises resources.
  • What sort of service does it provide: Storage
  • Non-cloud equivalent: Hard drive
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes

    Amazon EFS is a fully managed service that requires no changes to your existing applications and tools, providing access through a standard file system interface for seamless integration.
    Amazon EFS is a regional service storing data within and across multiple Availability Zones (AZs) for high availability and durability.

  • Related Services EC2

Amazon S3 Glacier

  • Description: Amazon S3 Glacier is a secure, durable, and extremely low-cost storage service for data archiving and long-term backup.

    It is designed to deliver 99.999999999% durability, and provides comprehensive security and compliance capabilities that can help meet even the most stringent regulatory requirements.

    • The Amazon S3 Glacier storage class provides three retrieval options to fit your use case.
      • Expedited retrievals typically return data in 1-5 minutes, and are great for Active Archive use cases.
      • Standard retrievals typically complete between 3-5 hours, and work well for less time-sensitive needs like backup data, media editing, or long-term analytics.
      • Bulk retrievals are the lowest-cost retrieval option, returning large amounts of data within 5-12 hours.
  • What sort of service does it provide: Storage
  • Non-cloud equivalent: ****
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes
  • Related Services S3

CloudFront

  • Description: Amazon CloudFront is a fast content delivery network (CDN) service that securely delivers data, videos, applications, and APIs to customers globally with low latency, high transfer speeds, all within a developer-friendly environment.

    CloudFront is integrated with AWS – both physical locations that are directly connected to the AWS global infrastructure, as well as other AWS services. CloudFront works seamlessly with services including AWS Shield for DDoS mitigation, Amazon S3, Elastic Load Balancing or Amazon EC2 as origins for your applications, and Lambda@Edge to run custom code closer to customers’ users and to customize the user experience.

  • What sort of service does it provide: Content Delivery (Network)
  • Non-cloud equivalent:
  • IaaS/PaaS/SaaS/ does not apply ****
  • AWS Managed? Yes
  • Related Services AWS Shield, EC2, S3, Elastic Load Balancing, Lambda@Edge

    CloudFront is highly effective when delivering both static and dynamic content to users located in different areas. With the help of edge locations, content is made available with low latency and customers gain improved performance. Accessing content via a VPN connection does not lend itself well to CloudFront as users appear to be in the same location.

RDS

  • Description: Amazon Relational Database Service (Amazon RDS) makes it easy to set up, operate, and scale a relational database in the cloud.

    It provides cost-efficient and resizable capacity while automating time-consuming administration tasks such as hardware provisioning, database setup, patching and backups.

  • What sort of service does it provide: Database
  • Non-cloud equivalent: ****
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes
  • Related Services EC2

Amazon RDS has 6 engines. These are • MySQL • Aurora • MS SQL • Oracle • ProgreSQL • MariaDB

DynamoDB

  • Description: Amazon DynamoDB is a key-value and document database that delivers single digit millisecond performance at any scale. It’s a fully managed, multi-region, multi-master database with built-in security, backup and restore, and in-memory caching for internet-scale applications.
  • What sort of service does it provide: key-value and document Database
  • Non-cloud equivalent: on-premises key-value and document database
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes, fully managed
  • Related Services EC2

Redshift

  • Description: Amazon Redshift is a fast, scalable DATA WAREHOUSE that makes it simple and cost-effective to analyze all your data across your data warehouse and data lake.
  • What sort of service does it provide: Database
  • Non-cloud equivalent: on-premises data warehouse
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes, fully managed
  • Related Services S3, EC2

    Redshift is ideal for data warehousing, performing back office analysis and aggregation of data from multiple sources.

Aurora

  • Description: Amazon Aurora is a MySQL and PostgreSQL compatible relational database ENGINE that combines the speed and availability of high-end commercial databases with the simplicity and cost-effectiveness of open source databases.
  • What sort of service does it provide: database
  • Non-cloud equivalent: on-premise database engine
  • IaaS/PaaS/SaaS/ does not apply PaaS
  • AWS Managed? Yes. Amazon Aurora is fully managed by Amazon Relational Database Service (RDS), which automates time-consuming administration tasks like hardware provisioning, database setup, patching, and backups.
  • Related Services EC2

    Aurora can read MySQL/PostgreSQL databases and is a natural upgrade point for MySQL/PostgreSQL that offers high availability and scalability

Elastic Load Balancing(ELB)

  • Description: Elastic Load Balancing (ELB) automatically distributes incoming application traffic across multiple targets, such as Amazon EC2 instances, containers, and IP addresses.
    • Elastic Load Balancing offers three types of load balancers:
      • Application Load Balancer is best suited for load balancing of HTTP and HTTPS traffic;
      • Network Load Balancer is best suited for load balancing of Transmission Control Protocol (TCP), User Datagram Protocol (UDP) and Transport Layer Security (TLS) traffic where extreme performance is required.
      • Classic Load Balancer provides basic load balancing across multiple Amazon EC2 instances and operates at both the request level and connection level.

        Elastic Load Balancing offers ability to load balance across AWS and on-premises resources using the same load balancer. This makes it easy for you to migrate, burst, or failover on-premises applications to the cloud.

  • What sort of service does it provide: Networking
  • Non-cloud equivalent: on-premise load balancer
  • IaaS/PaaS/SaaS/ does not apply: SaaS
  • AWS Managed? Yes
  • Related Services EC2, ECS

Auto Scaling

  • Description: Amazon EC2 Auto Scaling helps you maintain application availability and allows you to automatically add or remove EC2 instances according to conditions you define.
  • What sort of service does it provide: Compute
  • Non-cloud equivalent: No
  • IaaS/PaaS/SaaS/ does not apply: SaaS
  • AWS Managed? Yes
  • Related Services EC2
