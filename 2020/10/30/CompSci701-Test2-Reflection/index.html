Impressive questions

Is C/N a metric for Alterability

I like this question because it helps me to examine:

  1. what exactly is a “metric”
  2. when are we “measuring” things and when are we just using heuristics
  3. definition of Alterability: we want to know the effectiveness and efficiency of making changes, not just the number of classes we worked on.

Counting FAN-OUT

I was not expecting this coming, but it was a thought-invoking question.

  1. Do I cound String?
    • It is a class. But a java library class
    • Counting it or not counting it does not change our assessment of coupling: it’s always available with Java any way.
    • As long as I am consistent.
  2. Same number of FAN-OUT, different coupling
    • depending on FinancialValue and depending on KiwiFinancialValue are both FAN-OUT of 1
    • depending on abstract FinancialValue allows flexibility in using different financial value and allows extention, but depending on KiwiFinancialValue tights us with the concrete implementation solidly.
    • FAN-OUT is not a “metric” of coupling either.
