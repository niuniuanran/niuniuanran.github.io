DynamoDB is a NoSQL key/value and document database for internet-scale applications.

Features

DynamoDB is:

  • Fully managed
  • multi-region
  • multi-master
  • durable database
  • built-in security
  • backup and restore
  • in-memory caching

Provides:

  • Eventual consistent reads(default)
  • Strongly consistent reads

All data is stored on SSD storage, and is spread across 3 different regions.

Specify whatever Read Capacity Units and Write Capacity Units you need, and you just get it without you tweaking anything.

Consistent Reads

When data gets written to DynamoDB, it takes time to update the copies across multiple regions. It is possible to have inconsistent data. You can choose the read consistency:

Eventual Consistent Reads (Default)

Reads are fast but not guaranteed to be consistent.
All copies of data eventually consistent within a second.

Strongly Consistent Reads

Guarantee of consistency, but higher latency (of within a second): have to wait until all results are consistent before result is returned.
