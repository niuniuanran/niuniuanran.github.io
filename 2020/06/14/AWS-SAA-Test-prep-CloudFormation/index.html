AWS CloudFormation: a templating language that defines AWS resources to be provisioned.
Automating the creation of resources via code.

Infrastructure as code

The process of managing and provisioning computer data centres (eg AWS) through machine-readable definition files rather than physical hardware configuration of interactive configuration tools (stop doing manual configuration)

Use Case

People pay a monthly subscription and we run a Minecraft server. They choose where they want and what size of server they want to run.
We can use their inputs and use an Lambda to create a new CloudFormation stack. We have a lambda to send them the email of their new Minecraft Server IP address and details.

Template Anatomy

Template Sections:

  • MetaData: Additional information about the template
  • Description: What the template does
  • Parameters: what inputs are allowed to pass to the template at runtime
  • Mappings: a lookup table. for example, mapping region ids to image ids in that region
  • Conditions: whether resources are created or properties are assigned
  • Transform: applies macros (like applying a mod which changes the anatomy to the custom)
  • Resources: resources to be created
  • Outputs: return value, e.g. the IP address of the new server created.

Only Resources are mandatory. CloudFormation Templates requires you to list at least one resource.

AWS Quick Starts

ROLLBACK_IN_PROGRESS

When CloudFormation encounters an error, it will roll back with ROLLBACK_IN_PROGRESS

Size limit

CloudFormation templates larger than 51,200 bytes (0.05MB) are too large to upload directly, and must be imported into CloudFormation via an S3 bucket

NestedStacks

helps you break up CloudFormation template into smaller reusable templates that can be composed into larger templates.
