CloudWatch: a collection of monitoring services for logging, reacting and visualizing log data.

CloudWatch is multiple services under one name. We have:

  • CloudWatch Logs: any custom log data, memory usage, rails logs, nginx logs
  • CloudWatch Metrics: metrics that are based off of logs, eg. memory usage
  • CloudWatch Events: trigger an event based on a condition
  • CloudWatch Alarms: triggers notification based on metrics which breach a defined threshold
  • CloudWatch Dashboards: create visualization based on metrics

CloudWatch Logs

Core service of CloudWatch: all other CloudWatch services are built on CloudWatch Logs.
Used to monitor, store and access log files.

Log Group

A collection of logs. Log files must belong to a log group

Log Stream

A log in a log group

Log lifecycle

By default, logs are kept indefinitely and never expire.

Most AWS services are integrated with CloudWatch Logs. Logging of services sometimes need to be turned on, or requires the IAM Permission to write into CloudWatch Logs.

CloudWatch Metrics

Represents a time-ordered set of data points.
CloudWatch comes with many pre-defined metrics. For example, EC2 per-instance metrics:

  • CPUUtilization
  • DiskReadOps
  • DiskWriteOps
  • DiskReadBytes
  • DiskWriteBytes
  • NetworkIn
  • NetworkOut
  • NetworkPacketsIn
  • NetworkPacketsOut

CloudWatch Events

Triggers an event based on a condition or on schedule.
Event Source(how to trigger the event) -> Targets(what to trigger)

Event Source

It could either be an Event Pattern or a schedule

Targets

You can select Target to invoke when an event matches your Event Pattern or when schedule is triggered.
The Target options are a long list. For example, Lambda function, SNS topic, SQS queue, Kinesis stream, Firehose delivery stream.

CloudWatch Custom Metrics and High Resolution Metrics

Using AWS CLI or SDK you can create and publish your own custom metrics.
When publishing a custom metrics, you can define it as either standard resolution or high resolution.
High Resolution Metrics:
Only available for custom metrics. You can track every:

  • 1s
  • 5s
  • 10s
  • 30s
  • 60s * n
    Higher the frequency, higher the cost.

CloudWatch Alarms

Triggers a notification based on metrics which breach a defined threshold.
