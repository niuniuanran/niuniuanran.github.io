CloudTrail: Logs API calls and actions on AWS services.
When you need to know who to blame.

CloudTrail Introduction

CloudTrail enables governance, compliance, operational auditing and risk auditing of your AWS account.

AWS CloudTrail is used to monitor API calls and Actions made on an AWS account.

Easily identify:

  • Where: Source IP address
  • When: Event Time
  • Who: User Identity, UserAgent
  • What: Region, Resource, Action

CloudTrail - Event history

By default already logging and will collect logs for 90 days via Event History.

If you need more than 90 days, create a Trail.
Trails are output to S3 and to not have GUI like Event History. To analyze a Trail you have to use Amazon Athena.

Trail Options - Important!

  • A Trail can be set to log to all regions
  • A Trail can be set to apply to all accounts in an Organization
  • You can Encrypt your Log file using Server side encryption via KMS (SSE-KMS)
  • We can ensure the integrity of the logs to see if they have been tampered by turning on Log File Validation

CloudTrail -> CloudWatch

You can send CloudTrail event to CloudWatch logs, but not the other way around.

Management Events and Data Events

Data Events

Data Events are for two services currently: S3 and Lambda. You can track actions such as: GetObject, DeleteObject, PutObject. Turned off by default

Management Events

Tracks management operations. Turned on by default. CAN’T be turned off
