Deliver web pages/content to users based on their geographical location, the origin of the webpage, and the content delivery server.
Can be used to deliver an entire website including static, dynamic and streaming.
Requests for content are served from the nearest Edge Location for the best possible performance.

CloudFront Components

Origin

Location of original files, can be S3 Bucket, EC2 instance, ELB, or Route53.

Edge Location

Position of the cache.

Distribution

A collection of Edge locations which defines how cached content should behave.

CloudFront Distribution

A Distribution is a collection of Edge Locations.
To set up your distribution:

  • Specify the origin
  • Select your Price Class (how many edge locations you’ll use)
  • There are two types of distributions:
    • Web (for websites)
    • RTMP (for steaming media)

Behaviours

Redirect to HTTPs, restrict HTTP methods, Restrict View Access, set TTLs (time to live. Content expires and refreshes after TTL)

Invalidations

Manually invalidate cache on specific files. Useful when you update your origin and want your updates immediately take effect for users.

Error Pages

Serve up custom error pages, e.g. 404

Restrictions

You can use Geo Restriction to blacklist or whitelist specific countries

Lambda@Edge

We use Lambda@Edge functions to override the behaviour of request and requests.
There are four available Lambda@Edge functions:

  • Viewer request: when CloudFront receives a request from a viewer
  • Origin request: before CloudFront forwards a request to the origin
  • Origin response: when CloudFront receives a response from the origin
  • Viewer response: before CloudFront returns the response to the viewer

Example user case:

  • use Cognito to control access to protected content
