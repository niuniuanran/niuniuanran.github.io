Timing of refactoring?

After looking at implementation one, I have more thoughts about when to refactor.
Before I think that big refactoring should happen immediately after I have implemented the code. I thought I should look at the code and think, what will need in the future, and how to prepare for them.
Now I think big refactoring might be more efficient before I need to extend or change.
Of course I still should review and improve my code continuously, giving a better name and better structure etc., but major refactorings, such as extracting a new inheritance hierarchy, will be much more efficient when I have a realistic direction of how the code needs to be changed or extended.
